# нюансы

1) умное хранилище - ставится на рельсы, 
длинна - один вагон,
можно приципить выход/вход конвеера(модуль),
объединяет вместимость с другими станциями (предел объеденения можно ограничить),
можно подключить к логистической сети чтобы считать содержимое
2) умная станция - ставится на рельсы как обычная станция,
имеет логистическую сеть как и обычная жд станция,
добавления - запрос разных ресурсов через входные сигналы,
имеет уникальный id, чтобы поезд вез ресурс именно к ней,
приоритет служит местом в очереди на привезение(если приоритет одинаковый, то поезда везур ресурсы в порядке очереди запроса),
еслть галочка отправлять/запрашивать через временное хранилище,
есть галочка - использовать как временное хранилище (поезда будут привозить определенные ресурсы только на эту станицю если у них стоит галка отправлять только на временное хранилище),
3) умные поезда - автоматически находятся в логистической сети планеты,
есть галочка на перевозку предметов только из временных хранилищ(не возьмут маршрут, если в временном хранилище нет ресурсов),
система находит ближайшую станцию/временное хранилище для удовлетворения потребности запроса станции запроса с подходящим условием и ближайший поезд по этому маршруту
поезда могут ехать не только по названию станции, но и по скрытому id (для механики надо)
4) пункт временного хранилища -  может хранить только определенный вид ресурса в большом количестве (x5 от стального сундука),
можно приципить выход/вход конвеера и объединить с другими хранилищами,
показывает количество и количество стаков ресурса(выходные сигналы, хз надо подумать),
5) модуль вход/выход с конвеера - заменяет манипулятор и имеет пропускную способность такую же как и у конвеера на входе,
имеет один вход (для конвеера) и выход (для умных хранилищ и временных), можно ограничить пропускную способность на 50(так будет лучше)
